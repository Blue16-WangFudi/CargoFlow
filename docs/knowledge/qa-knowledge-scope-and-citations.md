# 智能问答知识库范围与引用策略

## 任务来源

- 任务：知识库范围与引用策略
- 来源：PRD FR-11 / PRD 9.4
- 验收标准：明确知识来源、更新方式、引用格式、不可回答范围和隐私边界。

## 目标

CargoFlow 智能问答服务面向货主、司机、调度员、仓库管理员和系统管理员提供物流规则问答与授权业务上下文问答。回答必须可追溯、可解释、可拒答，且不得绕过接口层权限策略。

一期只支持两类问题：

| 问题类型 | 示例 | 数据来源 | 响应要求 |
| --- | --- | --- | --- |
| 物流规则与平台使用 | "偏航告警怎么判定？"、"司机如何确认调度指令？" | 受控知识库文档 | 必须给出引用来源；不确定时说明无法确认。 |
| 本人授权业务上下文 | "我的货物现在到哪了？"、"这票运输有哪些告警？" | API 授权过滤后的业务数据 | 必须先按角色和对象权限过滤，再进入检索和生成。 |

## 知识来源

一期知识库只允许纳入以下来源。未列入来源不得直接进入检索索引。

| 来源 | 范围 | 归口 | 进入索引条件 |
| --- | --- | --- | --- |
| PRD 文档 | 业务角色、功能规则、状态流转、验收场景 | 产品负责人 | 文档位于 `docs/`，并经过 PR review。 |
| 运营规则 FAQ | 常见运输规则、异常处理口径、客服标准答复 | 运营负责人 | 有版本号、发布日期和负责人。 |
| 系统使用说明 | 角色工作台、告警处理、车辆绑定、司机上报等操作说明 | 产品和交付负责人 | 与当前上线功能一致。 |
| 接口与数据字典摘要 | 业务对象定义、字段含义、枚举状态 | 工程负责人 | 来源于已合入的 API 契约、迁移或数据模型文档。 |

业务数据库中的货物、车辆、运输任务、位置、告警、调度指令、状态上报、通知、审计和问答记录不是公共知识库内容。它们只能作为用户请求期间的授权业务上下文使用，并且不得被写入共享知识库索引。

## 更新方式

知识库更新采用 PR 驱动流程：

1. 在 `docs/` 下新增或修改源文档，写明来源、责任人和适用版本。
2. PR 中说明影响的问答范围和需要重新索引的文档路径。
3. `scripts/check.sh` 通过后合入主干。
4. RAG 索引任务只从主干已审核文档生成向量，记录文档路径、标题、段落锚点、版本和索引时间。
5. 索引完成后保留上一版本索引，直到 smoke 问答验证通过。

下列变更必须触发重新索引：

| 变更 | 是否重新索引 | 说明 |
| --- | --- | --- |
| 物流规则、告警阈值、角色权限、状态流转变更 | 是 | 这些内容会直接影响回答口径。 |
| UI 文案、页面说明、操作路径变更 | 是 | 防止回答过期操作步骤。 |
| 纯格式调整、错别字修复 | 视情况 | 不改变语义时可延后到下一次批量索引。 |
| 业务数据新增或状态更新 | 否 | 业务上下文由实时授权查询提供，不进入知识库索引。 |

## 引用格式

每条基于知识库的回答必须包含 `sources` 数组。业务上下文回答如果使用了运输任务、货物或告警数据，也要包含可审计的业务引用，但不能暴露未授权对象。

推荐响应结构：

```json
{
  "answer": "偏航告警按计划路线与实时位置偏差触发，具体阈值以当前告警规则配置为准。",
  "sources": [
    {
      "type": "knowledge_doc",
      "title": "智慧物流 PRD 需求文档",
      "path": "docs/智慧物流_PRD需求文档.md",
      "section": "FR-04 异常报警",
      "version": "v1.0"
    }
  ],
  "business_refs": [
    {
      "type": "transport_task",
      "id": "task_123",
      "display": "运输任务 task_123"
    }
  ],
  "confidence": "high"
}
```

引用规则：

- `sources` 只引用用户有权查看或公共可见的知识文档。
- `business_refs` 只返回用户已授权对象的稳定业务标识或脱敏展示名。
- 当回答由多段文档共同支持时，按贡献顺序列出来源。
- 不得伪造引用；没有可靠来源时必须拒答或提示无法确认。
- 前端展示引用时使用文档标题、章节和版本，不展示内部向量片段 ID。

## 不可回答范围

智能问答遇到以下情况必须拒答、转人工或提示无法确认：

| 场景 | 处理方式 | 示例回答口径 |
| --- | --- | --- |
| 请求非本人或未授权业务数据 | 拒答 | "我不能查看你未授权的货物或运输任务。" |
| 查询权限策略、密钥、内部配置或安全绕过方式 | 拒答 | "该信息不在可回答范围内。" |
| 要求修改业务状态、关闭告警、下发指令 | 引导到业务功能，不直接执行 | "请在调度工作台处理该告警。" |
| 知识库没有覆盖或来源冲突 | 提示无法确认 | "当前知识库无法确认该规则，请联系运营或调度确认。" |
| 法律、保险理赔、合同争议等专业判断 | 转人工 | "该问题需要人工确认，系统不能给出最终判断。" |
| 用户输入包含明显恶意注入或要求忽略权限 | 拒答并记录 | "无法处理该请求。" |

## 隐私边界

权限裁决必须发生在 API 查询构造和业务服务层，RAG 模型不得作为权限判断来源。

| 角色 | 可检索业务上下文 | 禁止暴露 |
| --- | --- | --- |
| 货主 | 本人货物、关联运输任务、车辆基础信息、授权告警、本人问答历史 | 其他货主货物、司机联系方式、系统内部备注。 |
| 司机 | 本人承运任务、调度指令、本人状态上报 | 其他司机任务、货主完整资料、后台告警规则配置。 |
| 调度员 | 授权范围内车辆、任务、告警、指令和通知记录 | 非管辖区域数据、系统密钥、未授权用户隐私。 |
| 仓库管理员 | 仓库车辆、设备、绑定关系和相关审计 | 货主跨仓数据、司机非任务相关信息。 |
| 系统管理员 | 系统管理和审计范围内数据 | 密钥明文、访问令牌、模型提示词中的敏感配置。 |

隐私处理要求：

- 检索前按角色、组织、货主、司机和任务绑定关系过滤候选业务记录。
- 向模型传入的业务上下文只保留回答所需字段，默认不传手机号、证件号、完整地址、密钥、令牌和内部备注。
- 问答记录保存用户问题、回答摘要、引用、会话时间、反馈和权限过滤结果，不保存未经授权的数据片段。
- 日志和错误追踪中不得记录完整模型上下文；需要排查时使用脱敏 trace。
- 共享知识库索引不得包含客户私有运输数据或个人敏感信息。

## 实现约束

- RAG 检索服务只接收已授权的 `principal`、`role`、`tenant_scope` 和业务对象约束。
- 检索服务返回候选文档片段时必须携带来源元数据，生成层不得删除这些元数据。
- 业务上下文检索和知识库检索分开执行，再由 API 服务合并上下文。
- 所有智能问答 API 响应都要落库形成问答记录，便于用户反馈、审计和后续验收。
- CI 至少运行 `scripts/check.sh`，保证策略文档链接和必备章节不被破坏。
